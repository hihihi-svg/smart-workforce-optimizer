🔄 Part 1: Kahn's Topological Sort

Concept: Orders task nodes linearly based on precedence constraints (DAG).
Logic: Scans tasks with 0 dependencies (in-degree), schedules them first, and reduces the dependency counts of neighboring nodes. Warns if a dependency loop (cycle) exists.
Complexity: O(V + E) where V=vertices (tasks) and E=edges (dependencies).

⏰ Part 2: Critical Path Method (CPM)

Concept: Calculates earliest and latest start/finish days for each task.
Logic: Forward pass finds earliest days; backward pass finds latest days. Slack = Latest Start - Earliest Start. Tasks with Slack = 0 are on the Critical Path (cannot be delayed).
Complexity: O(V + E).

🧮 Part 3: Employee Suitability Scoring

Concept: Scores fit penalty for each task-employee cell (lower is better).
Logic: Uses a Trie (prefix tree) to lookup skills in O(L). Scores based on matching skills, experience, and workload. Keeps best candidate profiles using a Min-Heap.
Complexity: O(T * N log K) where T=tasks, N=candidates, K=Heap capacity.

⚡ Part 4: Hungarian Algorithm

Concept: Pair tasks with employees to minimize overall penalty sum.
Logic: Normalizes the cost matrix using row/column minimum subtractions to expose zero-cost cells. Finds the optimal independent matching. Guarantees global minimum cost.
Complexity: O(N³) where N is candidate pool size.

🔄 Kahn's Topological Order Calculation

💡 What is this? Topological Sort orders tasks so that no task is scheduled before its predecessors are completed. Kahn's Algorithm does this by finding start tasks with 0 dependencies (in-degree), adding them to the queue, and removing their edges to reduce neighbor dependency counts.

💡 What is this? CPM calculates the earliest start (ES) and latest start (LS) days for each task. The Slack (Float) is the buffer time (LS - ES). Tasks with 0 Slack are critical; delay there will delay the entire project.
